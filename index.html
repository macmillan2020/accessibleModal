Avoid Keyboard Traps

WebAIM Success Criteria 2.1.2 requires that "Keyboard focus is never locked or trapped at one particular page element. The user can navigate to and from all navigable page elements using only a keyboard."

However...

There are cases when a keyboard trap is desirable. Think, for instance: the modal window.

Normally, when a modal window is displayed on screen, we don't want users able to access anything else.

Head's Up!

When this modal is open, the user should not be able to tab to any other content within the browser.
